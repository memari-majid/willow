# Evidence — for the SME

> SME, this folder is where you **show your work**. List the sources
> behind every claim, technique, or framework Willow draws from.
>
> The developer didn't put real citations in this project — those
> are yours. We can pre-fill the file structure; we can't pre-fill
> what science you're standing on.

## Why this matters

When a user (or a reviewer, or a regulator, or a journalist) asks
*"on what basis does this companion suggest box-breathing?"*, the
answer should not be "the developer thought it sounded calming". It
should be a citation in `references.md`.

The bot reads `references.md` along with the technique files, so it
will not invent provenance — it will only refer to the science that
**you** have written here.

## Files

- [`references.md`](./references.md) — every source you're drawing on
- [`glossary.md`](./glossary.md) — clinical terms the bot may use
  correctly (definitions in your words)
